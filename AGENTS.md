# Project Principles

- Run this project in low-memory mode by default. The production deployment uses free Render, where cumulative peak memory above 500 MB can restart the process, so prefer memory-bounded implementations, streaming/batching over bulk loading, small caches, and avoiding background work that grows resident memory.
- Minimize unnecessary Turso reads and writes. The free Turso plan has monthly limits of 500 million row reads and 10 million row writes/deletes, so avoid redundant queries, broad scans, repeated polling, noisy persistence, and write amplification. Prefer targeted queries, pagination, caching where memory-safe, idempotent updates, and batching only when it reduces total row operations.
- Do not expose backend implementation details in client-facing pages, prompts, errors, or other user-visible messages. Client-facing text must not reveal the database technology, database names, backend account names, email addresses, service providers, internal endpoints, credentials, environment variables, file paths, or infrastructure details. Keep messages business-focused, with safe next-step guidance when needed.
- Treat this directory as the working project folder. After each project update, sync the program files to the separate GitHub submission folder at `C:\Users\zhuxd\myapi`. Before the first sync, do a full comparison to avoid omissions. Do not sync local-only dependencies, caches, secrets, or runtime data unless explicitly requested.
